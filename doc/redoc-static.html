ユーザ Webhook

user_idの指定に従って、ユーザ情報を返してください。

query Parameters
target
required
any

webhookの識別子です。user固定です。

user_id
any

プロジェクトメンバーのIDです。未指定の場合は全ユーザを返してください。

Responses

application/json
{
  • "status": "success",
  • "userList": [
    • {
      • "id": "123",
      • "name": "taro tanaka",
      • "redmine_user_id": "1",
      • "mail_address": "taro.tanaka@gmail.com"
      }
    ]
}

カレンダー反映済みイベント取得 Webhook

calendar_idで指定されたカレンダー情報を取得します。

query Parameters
target
required
any

webhookの識別子です。calendar固定です。

calendar_id
any

カレンダー情報の識別子です。未指定の場合は全チケット情報を返してください。

from
required
any

カレンダーから取得する期間の開始日(yyyy-mm-dd形式)

to
required
any

カレンダーから取得する期間の終了日(yyyy-mm-dd形式)

version_name
required
any

対象バージョン名(PMHで登録している場合、バージョン名が予定のタイトルに付与されています。)

Responses

Response samples

  • 200
Content type
application/json
{
  • "status": "success",
  • "calendarList": [
    • {
      • "title": "ver.1.2.3_キックオフ",
      • "start": "2024-01-05 12:00:00",
      • "end": "2024-01-05 13:00:00",
      • "all_day_event_flag": "true"
      }
    ]
}

カレンダーにイベントを反映 Webhook

リクエストの内容をもとにカレンダーにイベントを反映させてください。

Request Body schema: application/json
target
string

webhookの識別子。calendar固定です。

calendar_id
string

カレンダーの識別子。

ymd
string <date>

そのイベントの日。yyyy-mm-dd指定

start
string <time>

そのイベントの開始時刻。hh:mm:ss指定。終日イベントの場合は指定されません。

end
string <time>

そのイベントの終了時刻。hh:mm:ss指定。終日イベントの場合は指定されません。

Responses

Request samples

  • Payload
Content type
application/json
{
  • "target": "string",
  • "calendar_id": "string",
  • "ymd": "2019-08-24",
  • "start": "14:15:22Z",
  • "end": "14:15:22Z"
}

Response samples

  • 200
Content type
application/json
{
  • "status": "success"
}

target
string

webhookの識別子。delete_event固定です。

calendar_id
string

カレンダーの識別子。

from
string <date>

検索対象期間(from)。yyyy-mm-dd指定

to
string <date>

version_name
string

バージョン名。(PMH経由でカレンダーにイベント登録している場合は、バージョン名がタスクに付与されています）
